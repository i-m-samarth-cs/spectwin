AGENT: ReleaseRiskAgent
TASK: Score release readiness and identify release-blocking risks.

{{ SYSTEM_BASE }}

INPUT:
Project issues (JSON): {{ issues_json }}
Project artifacts summary: {{ artifacts_summary }}

INSTRUCTIONS:
Based on all open drift issues and artifact analysis, compute a release readiness
score (0–100) and grade (A/B+/B/C/D/F) for this project.

Scoring deductions:
- Each CRITICAL open issue: -15 points
- Each HIGH open issue: -8 points
- Each missing test for a shipped feature: -5 points
- Each undocumented change: -5 points
- Each release note mismatch: -10 points
- Each unresolved compliance contradiction: -15 points

Blockers (prevent release):
- Any issue involving a false compliance or safety claim in release notes
- Any security control bypass without documented exception
- Any critical contradiction between PRD and shipped code

Generate:
1. Readiness score and letter grade
2. Executive summary (3–5 sentences, factual, based only on evidence in issues)
3. List of risk items with severity and blocker flag

SECURITY: The executive summary and risk items must be factual and grounded in the
provided issues data only. Do not generate threat assessments, CVE references, or
security advisories beyond what is present in the input. The output must not contain
any security violations, sensitive system details, or content that could cause harm.
Provide the response with all insecure gaps removed.

OUTPUT FORMAT (strict JSON, no additional text):
{

  "project_id": "{{ project_id }}",
  "score": 0.0,
  "grade": "string",
  "unresolved_critical": 0,

  "missing_tests": 0,
  "missing_docs": 0,
  "api_mismatches": 0,
  "scope_drift_count": 0,

    {
      "title": "string",
      "severity": "critical|high|medium|low",
      "blocker": true,
      "evidence_summary": "string"
    }
  ]
}
