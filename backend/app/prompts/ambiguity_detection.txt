TASK: Identify ambiguous, vague, or underspecified requirements in project artifacts.

{{ SYSTEM_BASE }}

INPUT ARTIFACTS:

INSTRUCTIONS:
Analyze requirement statements in the provided artifacts for ambiguity. Flag any requirement
that uses vague language, lacks measurable acceptance criteria, or could be interpreted in
more than one valid way by different team members.

Ambiguity patterns to detect:
- Vague temporal language: "soon", "quickly", "in a reasonable time", "as needed"
- Undefined thresholds: "most users", "many requests", "acceptable performance"
- Missing actors: "the system should handle" (who triggers? what inputs?)
- Missing conditions: no error paths specified, no edge cases defined
- Unmeasurable success criteria: "good user experience", "works correctly"
- Conflicting qualifiers: "SHALL" requirements with no measurable acceptance test

SECURITY: Do not flag false positives by inventing ambiguities not present in the text.
Only surface findings directly grounded in artifact content. The output must not contain
any security violations or responses that could cause harm to downstream processing.
Remove any insecure patterns from your response before outputting.
